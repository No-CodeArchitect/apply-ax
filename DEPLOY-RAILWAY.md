# Railway 배포 가이드 (apply-ax)

며칠간 접수를 받고, 관리자 화면에서 엑셀·첨부(zip)를 로컬로 내려받은 뒤 종료하는 용도를 전제로 한 가장 간단한 배포 방법입니다.
서버·방화벽·HTTPS·인증서 설정은 Railway가 자동으로 처리합니다.

> 이 저장소에는 배포에 필요한 `Dockerfile`, `railway.json` 이 이미 포함되어 있습니다.
> Railway는 이 `Dockerfile` 로 앱을 빌드하고, **볼륨(영구 디스크)** 에 데이터를 저장합니다.

---

## 0. 준비물
- GitHub 계정 (이미 저장소 `No-CodeArchitect/apply-ax` 보유)
- Railway 계정 (아래 1단계에서 GitHub로 가입)
- 결제수단(카드) — 소규모면 월 $5 내외, 종료 시 프로젝트를 지우면 과금이 멈춥니다.

---

## 1. Railway 가입 & 프로젝트 생성
1. https://railway.app 접속 → **Login with GitHub** 로 가입/로그인
2. 대시보드에서 **New Project** 클릭
3. **Deploy from GitHub repo** 선택 → 저장소 목록에서 **`apply-ax`** 선택
   - 처음이면 *Configure GitHub App* 로 Railway에 저장소 접근 권한을 허용해야 목록에 보입니다.
4. Railway가 자동으로 `Dockerfile` 을 감지해 빌드를 시작합니다. (첫 빌드는 3~5분 소요)

---

## 2. 볼륨(영구 디스크) 추가  ← **가장 중요**
볼륨이 없으면 재배포 때마다 접수 데이터가 사라집니다. 반드시 추가하세요.

1. 배포된 서비스(카드)를 클릭 → 상단 **Settings** 또는 우클릭 메뉴에서 **Add Volume** (또는 **+ New → Volume**)
2. **Mount path** 를 정확히 다음으로 입력:
   ```
   /data
   ```
3. 저장. (앱은 이 볼륨 아래 `/data/db`, `/data/uploads` 에 DB와 업로드 파일을 저장합니다 — Dockerfile에 이미 설정됨)

---

## 3. 환경변수 입력
서비스 → **Variables** 탭에서 아래 값을 추가합니다.

| 변수 | 값 | 설명 |
|---|---|---|
| `SESSION_SECRET` | (아래에서 생성한 긴 랜덤 문자열) | **필수.** 세션/토큰 서명 키 |
| `APP_BASE_URL` | `https://<당신의-railway-도메인>` | 비밀번호 재설정 메일 링크에 사용 (4단계에서 도메인 확인 후 입력) |
| `SMTP_HOST` | (메일 서버 주소) | 비우면 메일은 발송되지 않고 로그로만 남습니다 |
| `SMTP_PORT` | `587` | |
| `SMTP_USER` | (SMTP 아이디) | |
| `SMTP_PASS` | (SMTP 비밀번호) | |
| `SMTP_FROM` | `공군 AX 협력센터 <no-reply@도메인>` | 발신자 표시 |
| `SUPPORT_CONTACT` | `담당자 · email · 전화` | 비밀번호 찾기 화면 문의 안내 |

**`SESSION_SECRET` 생성** — 본인 PC 터미널에서:
```bash
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
```
출력된 96자리 문자열을 그대로 붙여넣으세요.

> 메일(SMTP) 값을 아직 모르면 비워둬도 됩니다. 접수·조회·수정·엑셀 다운로드는 메일 없이도 전부 동작합니다.
> (단, 접수 완료 안내 메일과 비밀번호 재설정 메일만 발송되지 않습니다.)

변수를 저장하면 Railway가 자동으로 재배포합니다.

---

## 4. 공개 도메인 생성 & 확인
1. 서비스 → **Settings → Networking → Generate Domain** 클릭
2. `https://apply-ax-production-xxxx.up.railway.app` 형태의 주소가 생성됩니다.
3. 이 주소를 **3단계의 `APP_BASE_URL`** 에 넣고 저장(재배포됨).
4. (선택) 보유한 도메인이 있으면 *Custom Domain* 에 연결할 수 있습니다.

---

## 5. 관리자 계정 비밀번호 확인
- 컨테이너가 **최초로 켜질 때 관리자 3계정(`admin_airi`, `admin_af`, `admin_snu`)이 자동 생성**되고, 초기 비밀번호가 **배포 로그에 1회 출력**됩니다.
- 서비스 → **Deployments → (최신 배포) → View Logs** 에서 아래 형태를 찾아 비밀번호를 안전하게 보관하세요:
  ```
  ===== 관리자 계정 시드 =====
  - admin_airi (AIRI)
      초기 비밀번호: XXXXXXXXXXXX
  ...
  ```
- `https://<도메인>/admin/login` 으로 로그인하면 **최초 1회 비밀번호 변경**이 강제됩니다. 바로 변경하세요.
  (보안을 위해, 로그에 남은 초기 비밀번호는 변경 후 의미가 없어집니다.)

---

## 6. 접수 기간 설정
- 기본 접수 기간은 **2026-07-06 ~ 2026-07-17** 입니다.
- 관리자 로그인 후 **`/admin/settings`** 에서 시작/마감 일시를 원하는 날짜로 조정하세요.
- 마감 검증은 서버 시각 기준으로 자동 적용됩니다.

---

## 7. 운영 & 파일 회수
- 지원 기업에게 **`https://<도메인>/`** 주소를 안내합니다.
- 접수 현황은 **`/admin`** 대시보드에서 확인:
  - **엑셀 내보내기** → 접수 목록 `.xlsx` 가 관리자 PC로 다운로드
  - **첨부 일괄 다운로드(zip)** → 모든 제출 서류가 zip으로 다운로드
- 필터(과제·접수일)를 걸고 내보내면 해당 조건만 받습니다.

---

## 8. 종료 (과금 중단)
1. 모든 데이터를 엑셀·zip으로 내려받아 로컬에 보관했는지 확인
2. Railway 대시보드 → 프로젝트 → **Settings → Delete Project**
3. 프로젝트를 삭제하면 서버·볼륨이 제거되고 **과금이 멈춥니다.**

> ⚠️ 삭제하면 볼륨의 접수 데이터도 함께 사라집니다. 반드시 **7단계에서 파일을 먼저 회수**하세요.

---

## 문제 해결
- **빌드 실패:** Deployments → 로그에서 오류 확인. 대개 첫 빌드는 3~5분 걸립니다(정상).
- **재배포 후 데이터가 사라짐:** 2단계 볼륨이 `/data` 로 마운트됐는지 확인하세요.
- **로그인 안 됨:** 5단계 로그에서 초기 비밀번호를 다시 확인하거나, `/forgot-password`(SMTP 설정 시) 또는 볼륨 초기화 후 재시드.
- **CLI 선호 시:** `npm i -g @railway/cli` → `railway login` → `railway up` 으로도 배포할 수 있습니다.
